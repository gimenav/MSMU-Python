Chapter 12: Functions

What is a function?

A function is a group of statements/block of code that performs a specific task when it is called.

In order to create a function, the programmer needs to:

  1. Define a function with the keyword "def" and the name of the function next to it.
    • A function name can not be a reserved Python keyword like, "sum" or "print".
  2. Create arguements/parameters(what is inside the parantheses), which functions don't necessarily needs to have. So, the parantheses can be empty.
  3. Once the function is defined, it has to be executed.
    • In order to do that, the programmer needs to call the function(function invocation) and then use the print function.

Once the function is defined, it has to be executed. In order to do that, the programmer needs to call the function(function invocation) and then use the print function.

What does the return statement do in functions?

The return statement is used to end the function and then it returns the end value of the statement.

What are type annotations?

Type annotations are used to show others who are reading the program what type the variables are. It also helps the people who are reading the program to know what the outputs and inputs will be of the program. But one thing to keep in mind, that typing type annotations will not affect the program, it's more so like a comment, in that it's for the reader to understand what is going on in the program.
