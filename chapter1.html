Chapter 1: Introduction

What is Programming?

Programming is the process of taking an algorithm and encoding it into a notation, a programming language, so that it can be executed by a computer. In other words, is a set of instructions send to be translated(by a compiler) into the language that the computer/machine understands, so it can be executed. Although many programming languages and many different types of computers exist, the important first step is the need to have the solution. Without an algorithm there can be no program.

What is an algorithm?

It is a step by step set of instructions used to solve a problem. For example, if you wanted to raise money to help people who live below the poverty line in Latin America, you need algorithms to figure out both the numbers of people that live in those conditions and how much money you need to raise.

High level languages vs. Low level languages

High level languages are programming languages that are easier to write, read, and be understood by humans. Plus, it is portable, which means that it works in different computers. However, since these languages are closer to the languages that we speak and write, they need to be processed and translated in the language that the computer understands. Some high level languages include: Python, C++ and Java.


Low level languages are easier to be interpreted by computers than by humans. These languages sometimes have letters and numbers, though others only consist of numbers. The computer processes the code faster, because the computer understands these languages, and doesn't need to be translated, like high level languages do. Some low level languages include: Assembly Language and Machine Code.

What is a token?

A token is the smallest individual unit in programming. There are four different types of token:

  • Keywords: These are a set of words that have a specific meaning or function in Python.
    • Example: if, import, while, not, for, else, and or.
  • Identifiers: Are names that the user assigns to entities, such as variables, functions, classes, modules or other objects.
  • These are the requirements to keep in mind, while creating an identifier:
    • It has to start with a letter or an underscore (_), not with a number or a symbol.
    • Identifiers can not have punctuation, @, % or the $ symbol.
    • Python is case sensitive, so if your identifier has an uppercase, and you call it with all lowercase, it would give you an error.
  • Literals: Is defined by data given to variables or constant. A couple of literal types include:
    • Numerical Literals: Unchangeable numerical types, like integers, floats and complexes.
    • String Literals: A set of characters inside quotation marks.
    • Operators:are special symbols used for arithmetic or logical computations.
